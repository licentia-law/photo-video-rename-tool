# 📘 사용 방법 (초보자용 가이드) – v6.9

이 문서는 **Photo & Video Rename + Copy Tool (v6.9)**을  
처음 사용하는 분도 쉽게 따라 할 수 있도록 작성된 안내서입니다.

👉 **명령어 입력 없이, 더블 클릭만으로 사용**할 수 있습니다.

---

## 1️⃣ 준비 사항 (필수)

아래 파일과 폴더가 **같은 위치**에 있어야 합니다.

```
RenamePhoto.bat
RenamePhoto.ps1
exiftool/
 └─ exiftool.exe
```

⚠️ **exiftool.exe가 없으면 실행되지 않습니다.**

---

## 2️⃣ 실행 방법

1. **RenamePhoto.bat** 파일을 더블 클릭합니다.
2. 폴더 선택 창이 차례대로 2번 나타납니다.
   - 📂 원본 사진/영상이 있는 폴더
   - 📂 정리된 파일을 저장할 폴더
3. 선택이 끝나면 자동으로 작업이 시작됩니다.

---

## 3️⃣ 실행 중 화면 설명 (v6.9 개선)

실행 중 콘솔 진행창에 다음 정보가 실시간으로 표시됩니다.

- 전체 진행률 (%)
- 처리 중 파일 수 / 전체 파일 수
- 복사(COPY) 개수
- 스킵(SKIP) 개수
- 오류(ERROR) 개수
- ⏱ **현재까지 소요 시간**
- ⏳ **남은 예상 시간 (ETA)**
- ⚡ **평균 처리 속도 (files/sec)**

⚠️ 작업이 끝날 때까지 창을 닫지 마세요.

---

## 4️⃣ 스킵(Skip) 및 중단 복구(Resume) 처리 방식 (v6.9 핵심)

### ✅ 기존 파일 고속 스킵
- 대상 폴더의 기존 파일을 **시작 시 한 번만 캐싱(HashSet)**
- 파일 존재 여부를 디스크가 아닌 **메모리에서 즉시 판단**
- 파일 수가 많아도 스킵 속도 저하 최소화

### 🔁 중단 복구 (Resume)
- 이전 실행 시 생성된 로그를 기준으로
- **이미 처리된 파일은 자동 스킵**
- 프로그램을 다시 실행하면 **중단 지점 이후부터 이어서 처리**

---

## 5️⃣ 로그 기록 방식 (권장 설정)

기본적으로 **성능을 우선하는 로그 정책**이 적용됩니다.

- `SUMMARY` (기본)
  - COPY는 개수만 집계
  - ERROR만 파일에 기록
- `FULL`
  - COPY / SKIP / ERROR 모두 기록 (디버그용)
- `OFF`
  - 파일 로그 미작성 (최고 속도)

👉 일반 사용자에게는 **SUMMARY 모드 권장**

---

## 6️⃣ 작업 완료 후

작업이 끝나면:

- 완료 안내 팝업 표시
  - 총 처리 파일 수
  - 평균 처리 속도
  - 총 소요 시간
- 결과 폴더 자동 열림
- 로그 파일 생성 (옵션에 따라)

예시:
```
rename_copy_log_20251216_135703.txt
```

---

## 7️⃣ 결과 폴더 구조

```
<DESTINATION>
├─ PHOTO
│  └─ YYYY
│     └─ MM
│        └─ DD
└─ VIDEO
   └─ YYYY
      └─ MM

```

---

## 8️⃣ 자주 묻는 질문 (FAQ)

### ❓ 원본 파일이 삭제되나요?
아니요.  
👉 **모든 작업은 복사(Copy) 방식**으로 진행됩니다.

---

### ❓ 촬영 시간이 같은 파일이 많아도 괜찮나요?
문제 없습니다.

- `IMG_####` 형식 → 번호 유지
- 그 외 파일 → 해시 기반 고유 ID 자동 생성

👉 파일명 충돌 없이 안전하게 처리됩니다.

---

## ✅ 요약

- 더블 클릭 → 폴더 2개 선택 → 자동 정리
- 대용량 사진/영상도 빠르게 처리
- 중단 후 재실행도 안전
- 초보자도 안심하고 사용 가능

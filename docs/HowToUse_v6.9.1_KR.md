# 📘 사용 방법 (초보자용 가이드) – v6.9.1

이 문서는 **Photo & Video Rename + Copy Tool (v6.9.1)**을  
처음 사용하는 분도 쉽게 따라 할 수 있도록 작성된 안내서입니다.

👉 **명령어 입력 없이, 더블 클릭만으로 사용**할 수 있습니다.

---

## 1️⃣ 준비 사항 (필수)

아래 파일과 폴더가 **같은 위치**에 있어야 합니다.

```
RenamePhoto.bat
RenamePhoto.ps1
tools/
 ├─ exiftool/
 │  └─ exiftool*.exe
 └─ magick/
    └─ magick.exe
```

- HEIC 처리를 위해 ImageMagick(`magick.exe`)이 필요합니다

---

## 2️⃣ 실행 방법

1. **RenamePhoto.bat** 파일을 더블 클릭합니다.
2. 폴더 선택 창이 차례대로 2번 나타납니다.
   - 📂 원본 사진/영상이 있는 폴더
   - 📂 정리된 파일을 저장할 폴더
3. 선택이 끝나면 자동으로 작업이 시작됩니다.

---

## 3️⃣ 실행 중 화면 설명

실행 중 콘솔 진행창에 다음 정보가 실시간으로 표시됩니다.

- 전체 진행률 (%)
- 처리 중 파일 수 / 전체 파일 수
- COPY / SKIP / ERROR 개수
- ⏱ 현재까지 소요 시간
- ⏳ 남은 예상 시간 (ETA)
- ⚡ 평균 처리 속도 (files/sec)

---

## 4️⃣ HEIC 파일 처리 방식 (v6.9.1)

- HEIC 파일이 발견되면 자동으로 JPG로 변환됩니다.
- 변환 품질: **quality 92**
- 변환 후 원본 HEIC의 메타데이터를 JPG로 복사합니다.
- 이후 JPG 파일은 일반 사진과 동일하게 처리됩니다.

---

## 5️⃣ 스킵(Skip) 및 중단 복구(Resume)

### ✅ 기존 파일 고속 스킵
- 대상 폴더의 기존 파일을 시작 시 한 번만 캐싱(HashSet)
- 디스크 접근 없이 메모리에서 즉시 판단

### 🔁 중단 복구 (Resume)
- 이전 실행에서 처리된 파일은 자동 스킵
- 재실행 시 중단된 지점 이후부터 계속 처리

---

## 6️⃣ 로그 기록 방식

- `SUMMARY` (기본)
  - COPY는 개수만 집계
  - ERROR만 파일 기록
- `FULL`
  - COPY / SKIP / ERROR 모두 기록
- `OFF`
  - 로그 파일 생성 안 함 (최고 속도)

---

## 7️⃣ 작업 완료 후

- 완료 안내 팝업 표시
- 평균 처리 속도 및 총 소요 시간 표시
- 결과 폴더 자동 열림

---

## 8️⃣ 결과 폴더 구조

```
<DESTINATION>
├─ PHOTO
│  └─ YYYY / MM / DD
└─ VIDEO
   └─ YYYY / MM / DD
```

---

## 9️⃣ 자주 묻는 질문

### ❓ 원본 파일이 삭제되나요?
아니요. 모든 작업은 **복사(Copy)** 방식입니다.

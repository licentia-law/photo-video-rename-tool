# 📘 사용 방법 (초보자용 가이드)

이 문서는 **Photo & Video Rename + Copy Tool**을  
처음 사용하는 분도 따라 할 수 있도록 **아주 쉽게 설명한 사용 설명서**입니다.

👉 **명령어 입력 없이, 더블클릭만으로 사용**할 수 있습니다.

---

## 1️⃣ 준비 사항 (필수)

아래 파일과 폴더가 **같은 폴더에 있어야 합니다**.

```
RenamePhoto.bat
RenamePhoto.ps1
exiftool/
 └─ exiftool.exe
```

⚠️ **exiftool.exe가 없으면 실행되지 않습니다.**  
(exiftool은 사진·영상의 촬영 정보를 읽기 위해 반드시 필요합니다)

---

## 2️⃣ 실행 방법

1. **RenamePhoto.bat** 파일을 더블 클릭합니다.
2. 폴더 선택 창이 2번 나타납니다.
   - 첫 번째: 📂 **원본 사진/영상이 있는 폴더 선택**
   - 두 번째: 📂 **정리된 파일을 저장할 폴더 선택**
3. 선택이 끝나면 자동으로 작업이 시작됩니다.

---

## 3️⃣ 실행 중 화면 설명

- 진행률 막대(%)가 표시됩니다.
- 현재 상태가 함께 표시됩니다.
  - 복사된 파일 수
  - 건너뛴 파일 수
  - 오류 파일 수

⚠️ **작업이 끝날 때까지 창을 닫지 마세요.**

---

## 4️⃣ 작업 완료 후

작업이 끝나면:

- 완료 안내 팝업이 나타납니다.
- 결과 폴더가 자동으로 열립니다.
- 작업 로그 파일이 생성됩니다.

### 로그 파일 예시
```
rename_copy_log_20251215_110827.txt
```

로그에는 다음 내용이 기록됩니다.
- COPY : 정상 복사
- SKIP : 이미 존재하여 건너뜀
- ERROR : 처리 중 오류 발생

---

## 5️⃣ 결과 예시 (폴더 구조)

```
EOSR7
 └─ PHOTO
    └─ 2025
       └─ 12
          └─ 15
             └─ 2025-12-15_11-08-27_0300_EOSR7.JPG
```

- 카메라별 폴더 자동 생성
- 사진 / 영상 분리
- 날짜별 자동 정리

---

## 6️⃣ 자주 묻는 질문 (FAQ)

### ❓ 원본 파일이 삭제되나요?
아니요.  
👉 **원본 파일은 절대 삭제되지 않습니다.**  
모든 파일은 **복사(Copy)** 방식으로 처리됩니다.

---

### ❓ 촬영 시간이 같은 파일이 많으면 덮어쓰지 않나요?
걱정하지 않으셔도 됩니다.

- IMG_번호가 있으면 그대로 사용
- 없으면 파일명 기반 고유 ID 자동 생성

👉 **파일명이 겹치지 않도록 안전하게 처리됩니다.**

---

### ❓ 카메라 정보가 없는 파일은 어떻게 되나요?
카메라를 알 수 없는 경우:

```
UNKNOWN
```

으로 표시되며, 정상적으로 정리됩니다.

---

## 7️⃣ 문제가 생겼을 때

1. 먼저 **로그 파일**을 확인하세요.
2. 문제가 계속되면:
   - 오류 화면 캡처
   - 로그 파일 첨부
   - GitHub Issue 등록

---

- 설정 필요 없음
- 사진·영상 초보자도 안전하게 사용 가능
